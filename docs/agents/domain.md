# Domain docs

This repository uses a single-context domain layout.

## Before working

- Read `CONTEXT.md` when it exists.
- Read relevant decisions under `docs/adr/` when they exist.
- Proceed silently when either is absent; create domain documentation only when actual terminology or decisions have been resolved.

Use canonical terms from `CONTEXT.md` in specs, tickets, tests, and implementation. Surface any conflict with an existing ADR explicitly.
